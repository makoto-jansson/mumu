#### Day 3-4: 灯台と波のアニメーションを作る

```

## 所要時間の目安
2〜3週間

### Week 3: Focus モード

#### Day 13: アプリの基本構造を作る

```
mumu_app_technical_spec.md を読んでください。

/app 以下にアプリのページ構造を作ってください:
- /app/page.tsx （アプリホーム画面）
- /app/focus/page.tsx
- /app/relax/page.tsx
- /app/layout.tsx （アプリ専用レイアウト — HPのヘッダー/フッターとは別のUI）

アプリゾーンの layout.tsx は:
- HPのヘッダー/フッターを非表示にする
- 代わりにアプリ専用のBottomNav（ホーム / My cup / ルーティン）を表示
- 背景は暗色（#0a0a0a）

アプリのホーム画面には:
- 小さい灯台アニメーション（Phase 1で作ったものを再利用）

#### Day 14-15: Focusモードの全画面を作る

```
仕様書の「3.2 Focusモード」に従って、全画面を実装してください:

### Week 4: Relax モード + データ保存

#### Day 18-19: Relaxモードの全画面を作る

```
仕様書の「3.3 Relaxモード」に従って実装してください:

1. RelaxSetup（開始前画面）
   - 「珈琲を淹れて、ゆっくり始めましょう。」（mumuの豆は売らない）
   - 気分を3択で選ぶ:「疲れた」「もやもや」「ぼんやりしたい」
   - 時間を選択（3 / 5 / 10分）

2. RelaxSession（呼吸セッション画面）
   - 画面中央に呼吸サークル（膨らむ・縮む）
   - 吸う4秒 → 止める4秒 → 吐く6秒
   - テキスト:「吸う」「止める」「吐く」が切り替わる
   - 灯台の光が呼吸に合わせて明滅するアニメーション
   - BGMにお湯を注ぐ音を薄くミックス（Howler.js）

3. 完了画面
   - 「お疲れさまでした。」+ 日替わりの一言
   - ペアリング提案: 音楽（Spotifyリンク）+ お菓子
   - 珈琲の豆提案はなし（開始前に淹れているはず）

useBreath hook を作成:
- 呼吸サイクル: inhale 4秒, hold 4秒, exhale 6秒（計14秒で1サイクル）
- 各フェーズを state で管理
- 呼吸サークルの scale 値を返す
```

#### Day 20: ペアリングマトリクスを作る

```
src/data/pairings.json を作成してください。

気分（tired / foggy / dreamy）× 時間帯（morning / afternoon / evening）×
季節（spring / summer / autumn / winter）の組み合わせで
ペアリング（音楽 + お菓子）を提案するデータです。

全組み合わせは 3×3×4 = 36パターン。

音楽は Spotify の実際のURLを使ってください。
落ち着いたアンビエント / ジャズ / クラシック系を中心に。

例:
{
  "tired": {
    "evening": {
      "spring": {
        "music": { "title": "Says", "artist": "Nils Frahm", "spotifyUrl": "https://open.spotify.com/track/..." },
        "snack": "ダークチョコレート"
      }
    }
  }
}

現在時刻から時間帯を自動判定するロジック、
月から季節を判定するロジックも実装してください。
```

#### Day 23: トップページの更新 + 公開

#### Day 26-28: Sparkの全画面を作る

```
