# mumu — プロダクト設計書

## 珈琲のある時間をつくるアプリ

**バージョン**: v2.0
**更新日**: 2026年4月12日
**プロジェクトオーナー**: 灯台の珈琲焙煎所mumu

---

## 1. プロダクト概要

### 1.1 コンセプト

「感性が、ふと、戻ってくるような瞬間を届ける」——mumuの珈琲のコンセプトを、アプリ体験として拡張する。

集中・リラックス・アイデア発想・感性の回復という日常の営みを支えるツールであり、その体験の中にmumuの珈琲が自然に溶け込む設計。コーヒー提案アプリではなく、「クリエイティブワーク＆ウェルビーイングのオールインワンアプリ」。

### 1.2 ターゲットユーザー

コーヒーガチ勢ではない。ものすごく詳しいわけじゃないけど美味しいから豆を挽いて飲む人、なんとなくカッコ良いと思って飲んでいる人。忙しい日常の中で「丁寧な時間」を取り戻したいと感じている層。

- 20〜30代の会社員・フリーランス・クリエイター
- 在宅ワーカー（集中とリラックスの切り替えに課題を感じている）
- SNSで丁寧な暮らし系のコンテンツを見ている層

### 1.3 デザインの方向性

- **参考UI**: Endel（ミニマル・アンビエント・美しいアニメーション）
- **モチーフ**: 灯台
- **ビジュアル表現**:
  - タイマー中：ドリップがポタポタと落ちるアニメーション
  - 灯台と波がゆらめいているアニメーション
  - シンプルな線画ベース。装飾を排した静かなデザイン
- **カラーパレット**: 落ち着いたトーン。灯台の光（暖色のアクセント）× 海・夜空（寒色のベース）
- **タイポグラフィ**: ミニマル。日本語は游ゴシック or Noto Sans JP。ウェイトは細め中心

## 2. モード設計

### 2.1 Focus brew — 集中する

**ユースケース**: 仕事に取りかかる / 企画書を書く / 勉強する

**科学的根拠**:
- ポモドーロテクニック（Cirillo, 1980s）
- 儀式効果（Harvard, Brooks 2016 — 作業前の儀式がパフォーマンスを向上させる）

**フロー**:

| ステップ | 内容 | 珈琲 |
|---------|------|------|
| 開始前 | 「珈琲を淹れてから始めましょう。カフェインが集中力の立ち上がりを助けてくれます。」淹れている間の呼吸ガイド（2分）。集中時間を選択（25 / 45 / 60分 or カスタム）。やることを一言メモ。 | 淹れる提案のみ。mumuの豆は売らない |
| 集中セッション | ミニマル画面。タイマー + アンビエント音（雨 / 焚き火 / カフェ / 静寂から選択）。アニメーション：ドリップがポタポタ落ちる or 灯台の光がゆっくり回転。珈琲の提案は一切なし | なし |
| 完了 → 休憩（5分） | 「お疲れさまでした。」+ 日替わりの感性プロンプト1つ（例：「窓の外を10秒見てみてください」）。もう1セットやるか、終了するか選択 | なし |
| 終了時 | 今日の集中サマリー。一言メモ（任意）| 2セット以上の時のみ：「珈琲豆、そろそろ足りてますか？」+ BASEリンク |

**デザインノート**:
- タイマー画面は灯台の光が静かに回転するアニメーション
- 時間経過に応じて背景の空が微妙に変化（朝→昼、昼→夕方など）
- 休憩時は波の音に切り替わる

---

### 2.2 Slow drip — リラックスする

**ユースケース**: 仕事の合間に一息 / 寝る前に落ち着きたい / 休日の朝

**科学的根拠**:
- Box breathing（米海軍SEALsが採用する呼吸法）
- 自律神経の副交感切り替え研究

**フロー**:

| ステップ | 内容 | 珈琲 |
|---------|------|------|
| 開始前 | 「珈琲を淹れて、ゆっくり始めましょう。淹れる時間そのものが、ひと息つくきっかけになります。」今の気分を3択で選ぶ：「疲れた」「もやもや」「ぼんやりしたい」。時間を選択（3 / 5 / 10分）| 淹れる提案のみ。mumuの豆は売らない |
| 呼吸セッション | 画面が呼吸リズムに合わせてゆっくり膨らむ・縮む。吸う4秒 → 止める4秒 → 吐く6秒。BGMにお湯を注ぐ音・豆を挽く音をさりげなくミックス | なし |
| 完了 | 「お疲れさまでした。」+ 日替わりの一言。「この時間をもう少し味わうなら」として音楽（Spotifyリンク）+ お菓子のペアリング提案 | 珈琲の豆提案なし（すでに手元にあるはず）。音楽とお菓子のみ |

**デザインノート**:
- 呼吸ガイドは灯台の光が呼吸に合わせて明滅
- 波が呼吸リズムに同期して寄せては返す
- ドリップがポタ…ポタ…とゆっくり落ちるアニメーション

---

### 2.3 Spark — アイデアを爆発させる

**ユースケース**: 企画・創作で行き詰まった時。新しい視点がほしい時。

**科学的根拠**:
- 注意回復理論（Kaplan, 1995）— 非日常的な視覚刺激が指向性注意を回復させ、拡散的思考を促進
- 強制連想法（de Bono）— ランダムな刺激と課題を強制的に結びつけることで思考の枠を壊す

**フロー**:

| ステップ | 内容 | 珈琲 |
|---------|------|------|
| テーマ入力（任意） | 「何に行き詰まってる？」一言入力。入力しなくてもOK | なし |
| STEP 1: Spark shuffle（2〜3分） | Tinderライクなカードスワイプ。カードに「画像 + 一言」が表示される。直感で「おっ」と思ったカードを右スワイプでキープ。3〜5枚キープしたら次へ自動遷移。テンポよく、ポップに、視覚的に楽しく | なし |
| STEP 2: Spark connect（2分） | キープしたカードが横一列に並ぶ。テーマカードも一緒に。「この中から2枚選んで、無理やりつなげてみて。」30秒タイマーで1つアイデアを書く × 3ラウンド。スピード重視、質より量 | なし |
| STEP 3: Spark pick | 3つのアイデアが並ぶ。ベストにスターを付ける。過去のSparkログにも保存 | なし |
| 完了 | 「お疲れさまでした。アイデアは寝かせると育ちます。」| 「珈琲豆、そろそろ足りてますか？次の一杯の準備はこちら。」+ BASEリンク |

**カードデッキ設計（300〜500枚を事前に用意）**:

- **写真カード**: 「水たまりに映る空」「錆びた自転車」「祭りの屋台の裸電球」「猫の後ろ姿」「割れた陶器」「木漏れ日」
- **言葉カード**: 「逆さまにする」「半分にする」「子どもに説明するとしたら」「100年後」「もし水だったら」
- **感覚カード**: 「酸っぱい」「ざらざら」「朝5時の空気」「雨上がりの匂い」「焦げた匂い」
- **対比カード**: 「静寂 vs 爆音」「直線 vs 渦巻き」「老人 vs 赤ちゃん」「砂漠 vs 深海」

**技術ノート**: カードデッキは静的JSON。シャッフルアルゴリズムで毎回違う組み合わせ。生成AI不要。

**デザインノート**:
- カードはシンプルな角丸。スワイプ時に軽い物理アニメーション
- BGMは軽快なアンビエント。Focusとは異なるエネルギッシュなトーン
- 強制連想タイムの30秒タイマーは視覚的にカウントダウン（緊張感）

---

### 2.4 Reclaim — 感性を取り戻す

**ユースケース**: 「自分が好きなものってなんだっけ？」を思い出すための時間。

**科学的根拠**:
- マインドワンダリング研究（Smallwood & Schooler, 2015）— 意図的に思考をさまよわせることで自己参照処理が促進される
- ジャーナリング効果（Pennebaker, 1997）— 書き出す行為が自己理解を深める

**フロー**:

| ステップ | 内容 | 珈琲 |
|---------|------|------|
| 開始前 | 「珈琲でも淹れて、ゆっくりどうぞ。」| 最も軽いトーンで淹れる提案。購入提案は絶対にしない |
| 問いかけジャーナリング（10〜15分） | 穏やかなBGMの中、2〜3分ごとに「問い」が1つずつ表示。答えはテキスト入力。例：「最近、時間を忘れて夢中になったことは？」「10代の頃、放課後に一番したかったことは？」「好きな映画のどのシーンが一番好き？それはなぜ？」 | なし |
| ふりかえりサマリー | 書いた回答のキーワードがワードクラウド的に表示。「今日のあなたのキーワード：音楽、海、絵を描くこと」。過去のReclaimログも閲覧可能。続けると「自分の好きなものマップ」が豊かに | なし |
| 完了 | 「お疲れさまでした。また来週。」以上。| 珈琲の購入提案は絶対に出さない。余韻を大事にする |

**問いかけの設計方針**:
- 「好き/楽しい/夢中」の記憶を掘り起こす方向に限定
- 悩みやネガティブな内省には向かわせない
- 50〜100問を事前に用意、日替わりでローテーション

**デザインノート**:
- 最も静かなモード。灯台が夜の海を照らすような、暗めの美しい画面
- 波の音がゆっくり流れる
- 問いの表示はフェードインでゆっくり現れる

**重要**: このモードでは一切豆を売らない。ここがmumuの良心であり、ブランドの核。ここで売ると世界観が壊れる。でもここで生まれた信頼が他のモードでの購入につながる。

## 3. ルーティン機能 — Weekly rhythm

### 3.1 概要

曜日ごとにどのモードを使うかを設定できる機能。ルーティンの可視化と習慣化を促す。

**科学的根拠**:
- 実行意図（Gollwitzer, 1999）— 「いつ・どこで・何をする」を事前に決めると実行率が2〜3倍に
- 習慣スタッキング（Clear, 2018）— 既存の習慣に新しい行動を接続する

### 3.2 UI設計（実装済み）

- 月〜日の7列タイムライン（`/app/routine`）
- 各曜日をタップするとモーダルが開き、時間帯（朝/昼/夜）とモードを選んで追加
- エントリーを長押しで削除
- `routineStore`（Zustand persist）で管理 → localStorage 永続化
- アプリホームの上部に「今日のルーティン」を表示（`TodayRoutine` コンポーネント）
- 各エントリーはタップで対応モードに直接遷移

### 3.3 通知設計

**未実装**。Web Push通知はサーバーサイドのVAPIDキー管理が必要なため見送り。
ブラウザが閉じている状態での通知は技術的に実現困難（特にiOS）。

### 3.4 ダッシュボード（My cup / `/app/dashboard`）— 実装済み

- 今日/今週/連続（ストリーク）の3サマリーカード
- 週間カレンダー（月〜日の7日間、モード色ドット表示）
- 今週のルーティン達成状況（設定=空丸・完了=塗りつぶし丸）
- モード別集計バー（今週のセッションをモードごとに）
- ルーティン設定へのリンク
- 常設EC導線「珈琲豆を探す」（BASEへのリンク）
- 「やれなかった」を責めない設計。数字のみ、ポジティブな文言なし

## 4. 珈琲のタッチポイント設計

### 4.1 3レイヤーの分離

珈琲に関する提案を3つのレイヤーに完全分離する。

#### レイヤー1: 淹れる提案（行為の提案。豆は売らない）

各モードの開始前に、珈琲を淹れることを軽く促す。効果の説明は1〜2文。mumuの豆の名前は出さない。スキップ可能。

| モード | 提案文 |
|-------|--------|
| Focus | 「珈琲を淹れてから始めましょう。カフェインが集中力の立ち上がりを助けてくれます。」 |
| Relax | 「珈琲を淹れて、ゆっくり始めましょう。淹れる時間そのものが、ひと息つくきっかけになります。」 |
| Reclaim | 「珈琲でも淹れて、ゆっくりどうぞ。」 |
| Spark | 淹れる提案なし（テンポ重視） |

#### レイヤー2: 豆の補充提案（終了後にさりげなく）

毎回ではなく、条件を満たした時のみ表示。

| タイミング | 条件 | 提案文 |
|-----------|------|--------|
| Focus終了時 | 2セット以上の時のみ | 「珈琲豆、そろそろ足りてますか？」+ BASEリンク |
| Spark終了時 | 毎回（さりげなく） | 「次の一杯の準備はこちら。」+ BASEリンク |
| 週次レポート | 毎週（AI生成文の最後） | 「来週のお供はこちらからどうぞ。」+ BASEリンク |
| Relax終了時 | — | 珈琲提案なし。音楽 + お菓子のペアリングのみ |
| Reclaim | — | 絶対に出さない |

**追加の表示条件**: 前回の補充提案から7日以上経過していること。

#### レイヤー3: 常設のEC導線（押し付けない）

| 設置場所 | 形式 |
|---------|------|
| アプリホーム画面の下部 | 「mumuの珈琲豆はこちら」+ 「HPに戻る」テキストリンク |
| ダッシュボード（My cup）内 | 「珈琲豆を探す」ボタン |
| HPトップ珈琲セクション | 豆カードを横並び表示（各カードからBASEへ直接遷移） |
| 設定 / Aboutページ | 「灯台の珈琲焙煎所mumuについて」+ ショップリンク |

**デザイン指針**: メインUIに溶け込むトーン、広告感ゼロ。テキストリンクかアイコン+テキスト程度。バナーやポップアップは使わない。

## 5. 週次レポート（AI実装）

### 5.1 概要

日曜夜にAIが生成する「あなたの1週間」レポート。数字の羅列ではなく、ユーザーのことをわかってくれている文章。

### 5.2 AIに渡すデータ（入力JSON）

```json
{
  "week": "2026-03-23〜2026-03-29",
  "sessions": {
    "focus": { "count": 5, "total_minutes": 150, "time_pattern": "morning" },
    "relax": { "count": 3, "avg_duration": 5 },
    "spark": { "count": 2, "ideas_saved": 4 },
    "reclaim": { "count": 1, "keywords": ["海", "絵を描くこと", "学生時代の図書館"] }
  },
  "previous_report_summary": "先週は集中多め、リラックス少なめだった",
  "season": "spring",
  "coffee_prompt_shown": 2,
  "streak": 3
}
```

### 5.3 出力のトーン

- mumuのAboutページと同じ、穏やかで親しみのある語り口
- 数字はさらっと触れる程度。主役は「あなたの1週間」への洞察
- 最後にmumuの豆をさりげなく提案（BASEリンク付き）

### 5.4 出力イメージ

**バランス型の週の場合**:
「今週は火曜と木曜に集中セッションを2回ずつ。水曜のお昼にリラックス、金曜の夕方にSparkでアイデア出し——バランスのいい1週間でしたね。先週よりリラックスの回数が1回増えていて、いいリズムです。来週も水曜のリラックスを続けてみてください。 — 来週のお供はこちらからどうぞ。」

**集中しすぎた週の場合**:
「今週は5日間で集中セッション8回、合計310分。先週の2倍近く集中していました。追い込みの週だったんですね。ただ、リラックスの時間がゼロだったのは少し気がかりです。来週はどこか1日、Slow dripで呼吸する時間を入れてみませんか？」

**Reclaimをした週の場合**:
「日曜のReclaimで「海」「絵を描くこと」「学生時代の図書館」というキーワードが出てきていました。どれも静かで、自分だけの空間を大事にする感性ですね。来週は一人の時間を少し多めに取ってみると、いい一杯に出会えるかもしれません。」

### 5.5 技術設計

| 項目 | 値 |
|-----|-----|
| モデル | Haiku 4.5 |
| 入力トークン | 約800〜1,200 |
| 出力トークン | 約300〜500 |
| 1回のコスト | 約$0.004（約0.6円） |
| 頻度 | 週1回（日曜夜にバッチ生成） |
| 月額（100人） | 約$1.6（約240円） |
| 月額（1,000人） | 約$16（約2,400円） |

プロンプトキャッシング + バッチAPI併用でさらに削減可能。

## 6. 技術構成（案）

### 6.1 フロントエンド

- **フレームワーク**: React（Next.js or Vite）
- **PWA対応**: ホーム画面追加、オフライン対応、プッシュ通知
- **アニメーション**: Framer Motion or CSS @keyframes
- **音声**: Web Audio API（アンビエント音、呼吸ガイド音）

### 6.2 バックエンド（最小構成）

- **MVP**: バックエンド不要。ローカルストレージでデータ管理
- **Phase 2以降**: Supabase（無料枠 → 月$25 Proプラン）
  - ユーザー認証
  - セッションデータの保存
  - 週次レポート生成のトリガー

### 6.3 AI連携

- **API**: Anthropic API（Haiku 4.5）
- **用途**: 週次レポート生成のみ（MVP）
- **呼び出し方法**: Supabase Edge Functions or Vercel Serverless Functions

### 6.4 ホスティング

- **Vercel**（無料枠で開始可能）
- **独自ドメイン**: mumu-coffee.com など（年間約1,500円）

### 6.5 段階的アプローチ

| Phase | 内容 | AIコスト |
|-------|------|---------|
| Phase 1 | 全モードをルールベースで実装。AIなし | ¥0 |
| Phase 2（ユーザー50人〜） | 週次レポートにHaiku追加 | 月¥500〜 |
| Phase 3（ユーザー200人〜） | レポートのパーソナライズ強化 | 月¥3,000〜 |
| Phase 4（収益化後） | Sparkの連想補助やチャット機能追加 | 月¥10,000〜 |

## 8. コンテンツ戦略（アプリ外）

### 8.1 noteとの役割分担

| プラットフォーム | 役割 | コンテンツ例 |
|---------------|------|------------|
| note | 読み物・ストーリー（SEO + note内発見性） | なぜmumuを始めたか / 焙煎への想い / 新しい豆との出会い日記 / 珈琲と感性のエッセイ |

- noteの記事末尾 → 「mumuアプリで体験する」リンク
